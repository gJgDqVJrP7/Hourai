# 進行
### 戦闘準備
- 各艦はランダムな位置に配置される。詳しくは船体の配置を参照すること。
- 全ての戦闘が始まる前に、砲塔や魚雷発射管を旋回し、展開可能な装甲を展開する
### 先制雷撃(1x2ターン)
- 雷撃・航空攻撃のいずれかを選択する。
- 大型魚雷は使用できない。
### 砲撃戦(3x2ターン)
- 砲撃戦開始時、魚雷発射管が魚雷を装填できる。
- ターン開始時に砲撃・航空攻撃のいずれかを選択する。
### 雷撃戦(2x2ターン)
- 雷撃のみが可能。
- 1ターン目は大型魚雷を使用できない。
### 砲雷戦
- ターン開始時に砲撃・雷撃・航空攻撃のいずれかを選択する。
  - 或いは、砲撃、雷撃、航空攻撃をround-robinに行う。どちらを先に行うかは事前に申告する。
- 砲撃・航空攻撃ターン終了時、魚雷の再装填を行うことができる。
# 勝利条件
- 射撃可能な主砲がなくなったターンの終了時、敗北する。
- 発射可能な魚雷をすべて消費し、砲撃での決着が見込めない場合や、指定したターンが経過した場合、以下のいずれかを行う。
  - 射撃可能な主砲の割合によって勝敗を決する。
  - 大型魚雷でない魚雷を設置によって装填し、試合を続行する。
  - 引き分けとする。
# 船の制限
- 現に南を向いている方向を艦首とみなす。
- TNTで破壊できないブロックを使用してはならない。
  - 但し、水・溶岩・金床は例外的に使用可能とする。
  - コマンドブロックも制限付きで使用できる。詳しくはコマンドブロックについてを参照すること。
- 1つ以上の「主砲」と任意の数の「副砲」を定めること。
- 「発射ボタン」を1つ以上定めること。
  - 現に発射ボタンを押さずして砲・機銃を射撃した場合、ルール違反である。
- 大型艦クラスの内、水流装甲展開に3か所以上の操作を要する場合、相手は水流装甲が展開される前にミサイル攻撃ターンを行うことができる。
  - このターンは進行や引き分け判定のターン数に含めない。
- 水深2m以下の地点に1つのプレイヤーの頭を「コア」として置く必要がある。
  - コアに接続されたブロックの内、水面上に出ているものが一つもない場合、「浸水」としてその時点で敗北する。
### 階級
- 大型艦クラスは、全長800m以内、全幅200m以内とする。
- 大型艦クラスは、全長400m以内、全幅100m以内とする。
- 小型艦クラスは、全長200m以内、全幅75m以内とする。
- 漁船クラスは、全長64m以内、全幅16m以内とする。
# 魚雷
- 魚雷は発射する前にルートブロック(スポンジ)の2マス上に装填されている必要がある。
  - ルートブロックの上に金床を配置することはできない。
  - ルートブロックには、一種の魚雷とそれと同様の外形をした魚雷しか装填できない。
- 発射時はx又はz軸にそった直進によって自艦の領域を離れなくてはならない。
## 命中判定
- 着弾目標の設定ができない場合は、敵艦の中心を指定位置とする。
- 魚雷は着弾時に魚雷の最も高い地点の上のブロックが水面である地点(要は、水深1mの地点)に着弾する。
- 魚雷は、着弾目標に対して自艦の全長の「誤差」が発生する。全長が21m以下の場合は、誤差が21になる。
- 旋回可能な魚雷発射管に装填された魚雷は、誤差を21にすることができる。
  - 旋回可能な魚雷発射管に装填された魚雷は、水面下を通らず直進によって自艦の領域を離れなくてはならない。
- 魚雷は、魚雷のいずれかのブロックの進行方向側、船体のあたり判定を有するブロックに接する位置、或いはそこから1m奥に設置することができる。
  - 進行経路にそのような位置がない場合、魚雷は命中しなかったと判断され、効果を及ぼさないような場所に廃棄する。
### 短魚雷
- 幅1、高さ3、総ブロック数16以下の魚雷を短魚雷とすることができる。
- 短魚雷は規定位置に直接設置することによって再装填することができる。
## 大型魚雷
- 幅3、高さ4、総ブロック100のいずれかを超過する場合は大型魚雷となる。
- 再装填不可能。
- カート式の場合、カートが装填済みの状態で保持する必要がある。
# 航空機
- 特攻機は水面より上に配置され、水面下を通らず直進によって自艦の領域を離れなくてはならない。
- 重力で落下するブロックをHAC化してはならない。
- 航空機は1ブロックでも破壊された場合、<撃破>として一切の操作ができなくなる。
- 特攻機の攻撃時、敵艦に密着するような位置はとれない。

# 細則
## ターン中に可能な行動
### 共通
- 以下を操作する。
  - 発射ボタンではないボタン
  - レバー
  - リピーター
  - コンパレーター
  - トラップドア
  - フェンスゲート
  - ドア
  - 魚雷・ミサイル以外のHAC
- ブロックのインベントリの中身を変更する。
  - TNTとTNTカートの数と配置は変更できない。
- アイテムを任意の位置に落とす。
- 感圧版を踏む。
- 以下のブロックを設置する。
  - レール
  - パワードレール
  - ディテクターレール
  - アクティベーターレール
### 砲撃ターン
- 発射ボタンを押し、十分な時間の後、耐水弾のための砂利・蜘蛛の巣を設置することができる。その後、ターンは終了となる。
### 雷撃ターン
- 魚雷一部或いはすべてを発射する。
### 航空攻撃ターン
- 航空機の一部または全部を発艦させる。
## 船体の配置
- 東西に200m離れた2点を選択し、これらを基準点と呼称する。
- 一隻目の艦は以下のように配置する。
  - 東側の基準点に、艦の最も西にあるブロックを合わせる。
  - X座標をそのままに、艦首のZ座標が基準点のそれとおなじになるようにずらす。
  - そこから南に、0m~(階級の最大全長-艦の全長)ランダムにずらす。
- 二隻目の艦は以下のように配置する。
  - 西側の基準点に、艦の最も東にあるブロックを合わせる。
  - X座標をそのままに、艦首のZ座標が基準点のそれとおなじになるようにずらす。
  - そこから南に、0m~(階級の最大全長-艦の全長)ランダムにずらす。
## 砲について
- TNTを発射可能な装置を砲と呼称する。
  - 主砲は砲でなくてはならない。
  - 指定のない場合、すべての砲は主砲として見做される。
  - 砲は順次射撃が望ましいが、一斉射撃でも構わない。
  - 砲に装填された砂が破壊された場合、<弾薬庫誘爆>として、当該砲を使用不能とする。
    - 射撃可能な主砲を失うことなく弾薬庫誘爆した砲に対する発射命令を停止できない場合、当該砲の砲口を石で塞ぐ。
      - 当該石がなくとも当該砲が発射不能となった時、速やかに当該石を撤去する。
    - 現にこれに背き発射した場合、ルール違反である。
- TNT以外のエンティティを発射可能な装置の内、砲ではないものを機銃とする。
  - 現に自らの雷撃ターンに機銃を射撃し、その投射物を相手領域に侵入させた場合、ルール違反である。
- 射撃可能な主砲とは、現に以下の条件を満たすこと。
  - 砲であること。
  - 主砲であること。
  - そのターンが自らの砲撃ターンであった場合は十分な時間の後再び発射ボタンを押すことによって、1つ以上のTNTを射出することができる状態であること。
  - そのターンが自らの砲撃ターンでない場合は、本ターン或いは次ターンに、1つ以上のTNTを中間地点以遠に射出することができる状態であること。
- 1ターンに同一の砲を3回まで発射することができる。
    - 但し、120弾頭以上或いは15クロック以上の砲については1回までとする。
## 魚雷について
- 大型魚雷はディスペンサーからTNTカートを出すことはできない。
- 大型魚雷は発射直前にTNTカートを装填することはできない。
- 同様の外形をした魚雷とは、魚雷外部からの撮影により映る部分のブロックの配置がすべて同一のものとする。但し、ブロックの同一性についてはIDのみを参照し、データ値は参照しない。
  - 例えば、赤の羊毛と青の羊毛は同一として扱われるが、赤の羊毛と石は同一として扱われない。
- 魚雷の予備弾は、雨ざらしに配置してはならない（試合中に雨ざらしになる分には構わない。）
## コマンドブロックについて
- 操作を単純化するため、限定的にコマンドブロックを使用してもよい。
  - コマンドブロックの使用許可は、大会運営者の負担を軽減するための措置である。紳士的に行うこと。
  - 求められた場合、艦内のすべてのコマンドブロックを申告し、その役割を明瞭に示す必要がある。
  - どの座標に配置されたとしても動作するように組むこと。
  - コマンドブロックで行う操作は、手動で行っても同様の結果をもたらすことが期待されなければならない。
    - つまり、以下のようなことはできない。
      - 超砂の様に手動で行うことができない行動
      - 敵艦にブロックを設置する等、手動で行うとルール違反になる行動
    - 以下のような利用を想定している。
      - 耐水弾の装填を毎度行う手間を省くため、砂設置用のコマンドブロックを設置する。
      - 水流装甲の展開に複数個所の操作が必要なため、その自動化。
  - 防御構造にコマンドブロックを利用してはならない。
  - 求められた場合、コマンドブロックは任意のタイミングで石に置換することができる。
- コマンドブロック付きトロッコは使用禁止とする。
## その他

  - commandBlockOutput = false
  - doDaylightCycle = false
  - doMobLoot = false
  - doMobSpawning = false
  - doTileDrops = false
- 爆発耐久値が600以上のブロック(金床を除く)が発生した場合、撤去する。
- 発射済みの魚雷・航空機は、当該魚雷・航空機の所有側が必要だと判断した場合自由に撤去できる。
- ルートブロックの上に金床が存在する場合、ターンの終了時に撤去する。
- めりこみ自爆に関しての補償はない。砂列にTNTディスペンサーを置かない等、各自で対策を取ること。
- 旧規定のミサイルは体当たり攻撃を行う航空機として扱われる。
- 旧規定の短魚雷は廃止された。
